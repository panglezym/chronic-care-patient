# 慢愈 · 患者端

**Chronic Care AI — Patient App Prototype**

> 面向糖尿病、高血压、心血管疾病等慢性病患者的智能健康管理 iOS App 原型，共 16 个页面，基于 Apple × Tesla 产品设计哲学构建。

---

## 预览

本仓库包含一个完整的高保真交互原型，在浏览器中直接打开即可查看所有页面。

```
chronic-care-patient.html   # 全部 16 屏，单文件原型
```

> **在线预览**：下载后用 Chrome / Safari 打开，无需任何构建步骤。

---

## 产品定位

慢愈帮助慢性病患者在日常生活中轻松完成健康管理闭环：

```
录入数据  →  AI 分析  →  医生互动  →  执行医嘱  →  月度复盘
```

**目标用户**：40–75 岁，已确诊糖尿病/高血压/心血管疾病，每 1–3 个月复诊一次的患者。

## 设计系统

### 设计哲学

> **Apple 的克制 × Tesla 的数据密度**

- **Apple**：颜色只在关键时刻出现；留白即设计；卡片去边框，用背景色区分层级；每个像素都有意图
- **Tesla**：把复杂状态做成一眼可读的仪表盘；数字大、对比强；绿/黄/红三色语义不产生歧义

### 颜色

```css
/* 品牌色 — 只有一个 */
--brand:  #e8622a;   /* 橙，用于强调、CTA、激活态 */
--brand2: #fff0e8;   /* 橙底浅色，用于卡片背景 */

/* 语义色 — 最小化 */
--go:     #34a853;   /* 绿：在范围内、已完成 */
--warn:   #f59e0b;   /* 琥珀：需关注 */
--stop:   #e53935;   /* 红：立即处理 */

/* 中性色 — 全部为暖灰 */
--ink:    #1a1714;   /* 主文字 */
--ink2:   #5c5550;   /* 次文字 */
--ink3:   #a09890;   /* 占位/标签 */
--paper:  #f8f4f0;   /* 页面背景 */
--card:   #ffffff;   /* 卡片背景 */
```

> 整套设计只用 **1 个品牌色 + 3 个语义色**，其余全为暖灰阶。去掉所有多余颜色。

## 核心交互说明

### 健康评分（首页 Hero）

```
综合评分 = 血糖达标率 × 35%
          + 血压达标率 × 30%
          + 用药依从率 × 25%
          + 数据完整度 × 10%
```

评分通过 SVG 圆环可视化，配合三条 Tesla 仪表盘式 gauge 横轴实时显示各维度。

### AI 助手上下文

AI 对话时，系统自动将以下数据注入提示词：

- 患者基本信息（姓名/性别/年龄/疾病列表）
- 近 14 天健康记录（血糖/血压/体重/心率）
- 当前用药方案及近 7 天依从率
- 近期医嘱和预警摘要

### 预警触发规则

| 级别 | 条件 | 触发动作 |
|------|------|----------|
| 🔴 高风险 | 收缩压 > 160 或 血糖 > 13.9 mmol/L | 即时推送 + 医生端预警 |
| 🔴 高风险 | 连续 3 天指标超标 | 生成 AI 原因分析 |
| 🟡 中风险 | 指标超目标未达高危 | App 通知 + 医生端列表 |
| 🟡 中风险 | 连续 7 天依从率 < 70% | 医生端随访建议 |

### 月度报告生成

每月 25 日自动触发，流程：

1. 汇总当月所有健康记录
2. 计算各指标达标率
3. 调用 LLM 生成报告标题（反映最显著的改善或问题）
4. 生成 2–3 条 AI 建议（针对最差指标）
5. 推送患者通知 + 医生端待审阅 +1

## 设计决策记录

<details>
<summary>为什么不用 Tab Bar 放 AI 入口？</summary>

AI 助手被设计为 FAB（悬浮操作按钮）而非独立 Tab，原因：

1. **AI 是工具，不是目的地**：用户不会"去 AI"，而是在需要时随时呼出
2. **Tab Bar 位置有限**：5 个 Tab 中放 AI 会挤掉更高频的"数据"或"提醒"
3. **FAB 始终可见**：无论在哪个页面，橙色 FAB 都在右下角，零额外导航成本

</details>

<details>
<summary>为什么首页不放快捷操作格？</summary>

早期版本有 4 个快捷操作格（录入血糖/血压/运动/体重），经过设计评审后移除，原因：

1. **视觉噪音**：4 个均等格子让视线没有焦点
2. **与 Tab Bar 重复**：数据录入可通过 Tab Bar → 数据页进入
3. **Tesla 原则**：核心信息（评分+指标）才是主角，操作入口退场

</details>

<details>
<summary>血压录入为什么独立设计？</summary>

血压不能复用血糖的单值录入设计，原因：

- 血糖是一个数字（如 6.8 mmol/L）
- 血压是两个相关数字（如 138/88），必须同时录入才有临床意义
- 用户输入习惯不同：血糖直接键入，血压需要"收缩压 → 确认 → 舒张压"的分步流程
- 实时反馈逻辑也不同：血糖只需对比一个阈值，血压需要同时判断两个值

</details>
